ARCHITECTURE RULE — READ THIS FIRST
🏗️ Every Screen Uses One App Shell

GioHomeStudio is a single-page application (SPA). The sidebar never disappears. The top URL bar never reloads. Clicking any nav item swaps only the right content panel. The sidebar, user info, credit balance, and status indicators are always visible. This is how Artlist, Linear, Notion, and every professional tool works.

Claude Code Note — Review Queue Connections
· Every AI generation (image or video) from any screen automatically lands here with status "Pending Review"
· "✓ Approve & Publish" → publishes to the selected platforms AND moves the item to /all-content with status "Published"
· "✕ Reject" → removes from queue. User can regenerate from the original project
· "✏️ Edit" → opens the source screen (Commercial Maker or Free Mode) for this item in edit mode
· "▶ Preview" → opens a full-screen video modal over the current page (does not navigate away)
· The "3" badge on the sidebar nav item decrements by 1 each time an item is approved or rejected

Claude Code Note — Modal Pattern
· This modal appears as an overlay over whichever screen the user is on. The page behind is dimmed but does not navigate away
· The "Model" dropdown list is populated from the MODEL_REGISTRY in registry.js — same data source that powers /ai-models screen

· "Generate Now" button: calls the generation API, shows a loading state, then closes modal and adds result to review queue
· The credit balance in the sidebar behind the modal updates immediately after generation completes
