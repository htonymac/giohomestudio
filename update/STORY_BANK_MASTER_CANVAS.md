## The 8 Layers of Story Bank

### Layer 1 — Story DNA
Every story has a core identity that never changes.
The system captures this first before any writing starts.

Fields:
- **Core premise** — one sentence: what happens and why it matters
- **Genre** — action / drama / comedy / thriller / children / motivational / horror / romance
- **Tone** — dark / light / hopeful / tense / funny / emotional / inspiring
- **Target audience** — children / teens / adults / business / general / global
- **Core emotion** — what should the viewer FEEL at the end
- **Platform target** — YouTube / TikTok / Instagram / Facebook / All
- **Story length** — short (under 3 min) / medium (3-10 min) / long (10+ min)
- **Production difficulty** — easy / medium / hard (AI estimates based on scene count + effects)

This DNA is read by every AI in the system.
When AI generates scenes, it uses the DNA.
When AI suggests music, it uses the DNA.
When AI brainstorms, it uses the DNA.

---

### Layer 2 — World Builder
For any story longer than one chapter, the world must be defined.

Fields:
- **Locations** — list of places in the story with descriptions
- **Time period** — present / historical / future / fantasy
- **World rules** — what is normal in this world (e.g. animals can talk, magic exists)
- **Culture notes** — any culture: Western, African, Asian, Latin, Middle Eastern, fantasy, sci-fi, historical
- **Recurring elements** — objects, symbols, sounds that appear across scenes

This prevents AI from generating inconsistent scenes.
Scene 3 should not describe a forest if the world is set in Lagos.

---

### Layer 3 — Character Arcs
Characters are not just names. They change.

For each character:
- **Starting state** — who they are at chapter 1
- **Wound** — what happened to them before the story starts
- **Want** — what they think they want
- **Need** — what they actually need
- **Arc type** — growth / fall / flat / transformation
- **Ending state** — who they are by the final chapter
- **Voice profile** — how they speak (formal / street / shy / aggressive / funny)
- **Linked to Character Voice system** — connects to their Piper voice in Hybrid Planner

AI uses this to write their dialogue correctly.
Bear does not speak like Dog. Dog does not speak like the narrator.

---

### Layer 4 — Conflict Engine
Every story lives or dies on its conflict.

- **Core conflict type** — man vs man / man vs nature / man vs self / man vs society
- **Conflict introduced at** — which chapter the main problem appears
- **Escalation points** — which chapters make things worse
- **Crisis point** — the darkest moment (usually 70-80% through the story)
- **Resolution type** — happy / tragic / open / bittersweet
- **Unresolved threads** — things left open for a sequel

AI uses this to know when to add tension, when to slow down, when to build music.

---

### Layer 5 — Chapter Engine
The actual writing layer. This is what Henry originally described.

Per chapter:
- Chapter number + title
- Raw user text (what user wrote)
- AI-expanded version (what AI wrote from user's text)
- User-edited final version (what actually gets used)
- Chapter type: opening / rising action / climax / falling action / resolution / epilogue
- Pacing tag: slow-burn / medium / fast / action-packed
- Dominant emotion: hope / fear / joy / sadness / anger / surprise / tension
- Scene count estimate (how many video scenes this chapter needs)
- Status: draft / expanded / edited / locked

Autosave every 30 seconds.
Version history — every save is kept. User can roll back.
User can reorder chapters by drag.

---

### Layer 6 — Scene Intelligence
This goes far beyond "5 seconds or 10 seconds."

Per scene:
- Scene ID: SC01, SC02...
- Scene type: action / dialogue / transition / reveal / emotional / establishing / montage
- Duration: 5s / 10s / 15s / 30s / 60s / custom
- Shot type: wide / medium / close-up / aerial / POV / tracking
- Dominant emotion: fear / joy / tension / sadness / excitement / calm
- Characters present: [list]
- Location: [from World Builder]
- Time of day: day / night / dawn / dusk
- Weather/atmosphere: clear / rain / fog / storm / golden hour
- Music mood: tense / uplifting / sad / epic / playful / silent
- SFX suggestions: [auto from scene description]
- Video prompt: auto-generated for Kling/Hailuo from all the above
- Estimated cost: based on duration + provider

When user clicks "Send to Hybrid Planner":
ALL of this data pre-fills the scene cards.
User never has to re-enter anything.

---

### Layer 7 — Story Health Dashboard
AI analyses the full story and gives a report card.

Scores (1-10):
- **Hook strength** — does the opening grab attention?
- **Pacing score** — does it move at the right speed?
- **Emotional variance** — does it go up and down or stay flat?
- **Character clarity** — are the characters distinct and consistent?
- **Conflict strength** — is there real stakes and tension?
- **Resolution quality** — does the ending feel earned?
- **Platform fit** — how well does it work for the target platform?
- **Production feasibility** — can this actually be made with available tools?

Overall score + specific improvement suggestions.
"Your opening is weak — add a hook in the first 5 seconds."
"Chapter 3 is too slow — combine with chapter 2."
"Dog has no clear motivation in chapter 4 — define what he wants."

---

### Layer 8 — AI Brainstorm (Ultra Low Cost)
Available at any tab, any time. Like a co-writer always present.

**Cost rules (non-negotiable):**
- Ollama phi3 → default (free, fast, good enough for brainstorm)
- Ollama mistral → for longer reasoning (free)
- Claude Haiku → only if Ollama is down (very cheap)
- Claude Sonnet/Opus → NEVER for brainstorm (too expensive)
- GPT-4o → NEVER (not needed here)

**What the brainstorm AI can do:**
- Suggest alternative plot directions
- Write a specific scene on request
- Rewrite a chapter in a different tone
- Generate 5 title options
- Suggest what music mood fits each chapter
- Identify plot holes
- Suggest character backstory
- Write dialogue for a specific character in their voice
- Suggest how to adapt story for different platforms
- Research cultural context (any culture — Japanese, Brazilian, Egyptian, French, American, African, Indian, etc.)
- Generate hook lines for social media captions

All brainstorm history saved per story.
User can refer back to old suggestions.

  expandedStory    String   @default("")
  genre            String   @default("")
  tone             String   @default("")
  audience         String   @default("")
  coreEmotion      String   @default("")
  platformTarget   String   @default("")

  difficulty       String   @default("medium")

  brainstormLog    Json?

## Story Flow — Moving Stories Between All Sections (Henry's Addition)

A story written in Story Bank must be movable to ANY section of GioHomeStudio.
The pipeline must work in all directions.

### Story Bank → Planners
User clicks "Send to Planner" and picks destination:
- → Hybrid Planner (scenes + durations pre-filled)
- → Movie Planner (chapters become acts, scenes pre-filled)
- → Children Planner (story + characters pre-filled)
- → Commercial Planner (core message + scenes pre-filled)

### Planners → Story Bank
User clicks "Save to Story Bank" from any planner:
- Current story/script saved back as a Story Bank entry
- Status set to "in_production"
- Linked back to the original story if one exists

### Story Bank → Drafts
User can mark any story or chapter as "Draft":
- Appears in a Drafts section on Story Bank homepage
- Drafts are private — not shown in main list
- User can resume drafts anytime
- Draft status: quick-idea / outline / in-progress / review / ready

### Cross-Planner Draft Flow
Every planner has a "Save as Draft" button:
- Saves current project state to drafts
- Draft visible from Story Bank → Drafts tab
- User can open draft from Story Bank and continue in the original planner

### Pipeline Status Tracking
Each story shows its journey:
```
Story Bank → Draft → Sent to Planner → In Production → Assembled → Published
```
This status updates automatically as the story moves through the system.

### API Routes for Pipeline
```
POST /api/story-bank/[id]/send-to-planner
  body: { plannerType: "hybrid" | "movie" | "children" | "commercial" }
  → creates project in target planner, returns redirect URL

POST /api/story-bank/[id]/draft
  → marks story as draft, saves current state

GET  /api/story-bank/drafts
  → returns all drafts across all planners

POST /api/[planner]/save-to-story-bank
  → saves planner project back to Story Bank
```
