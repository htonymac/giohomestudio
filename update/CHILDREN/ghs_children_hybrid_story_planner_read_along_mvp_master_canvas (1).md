---

2. Main Product Rule — Non-Negotiable Sync

Text must rhythm with narration

This is non-negotiable.

The highlighted text must align with the narration exactly enough for learning use.

That means:

when narration reads a word, that exact word must be the active highlighted word

when narration reads a phrase, the visual reading state must match the spoken phrase

when narration moves to the next word or line, the text highlight must move with it

the image or visual support must not fight the reading rhythm


This is the core educational quality rule.

This must never happen

narration says “cat” while “sat” is highlighted

narration moves ahead but the text stays behind

image changes before the spoken word is complete

music becomes so loud that the child cannot follow the word rhythm


Product rule

Read-along synchronization is a first-class product requirement, not a nice extra.

# A. Hybrid Reading Architecture Support Layer

This section supports the hybrid-first principle already defined in the master.

# B. Read-Along Timing Map Structure

Supports Section 16 — Read-Along Timing Engine

Each word or line should generate a timing object.

Suggested structure:

Timing Unit Example

Word: "Cat"
Start Time: 0.00 sec
End Time: 0.82 sec
Highlight Window: 0.00–0.82 sec
Pause After: 0.40 sec

Sentence Example:

"The cat sat."

Word Timing:

The — 0.00–0.30
cat — 0.30–0.85
sat — 0.85–1.40

Pause — 1.40–1.80

This structure supports:

accurate highlighting

visual pacing

music rhythm protection

child reading flow stability

# F. Music Safety Layer Support

Supports Section 13 — Music Rules.
